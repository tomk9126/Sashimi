    Introduction

    Sashimi provides an interface to record, track, and export race scoring data for time-based sporting events, such as a school Swimming Carnival.
    Sashimi streamlines the process of selecting and ranking candidates in a race, with support for editing several meets (Carnivals) at once.

    Structure

    ‘Carnival’
    A Sashimi Carnival is a race meet, storing race, athlete, and scoring data as well as its own attributes The below diagram visualises this:
    {Structure Chart}
    A carnival is stored as a single file, containing all athletes, events, and score data. Multiple carnivals may be open in Sashimi at a time.

    ‘Event’
    An event is defined by a criteria: Gender (If applicable), Age Group (If applicable), and event name. Once scored, if will carry time and ranking data that may be viewed and/or exported.

    ‘Athlete’
    An athlete is given a name, date of birth, and gender. This allows the person to compete in relevant events.
